# Nomura-Trade-Analytics-Dashboard

🏦 Nomura Trade Analytics Dashboard – An interactive Power BI dashboard designed to analyze and monitor investment banking trade operations, profitability, counterparty performance, trade lifecycle activities, risk exposure, and operational trade breaks using a simulated trading dataset for analytical and visualization purposes.

**2. Short Description / Purpose:**

The Nomura Trade Analytics Dashboard is a visually interactive Power BI report developed to simulate real-world investment banking and middle-office trade analytics. The dashboard helps analyze trade volumes, P&L performance, asset-class risk exposure, counterparty activity, trade status monitoring, and reconciliation breaks.

The primary purpose of this project is to demonstrate business intelligence, financial analytics, and dashboard storytelling skills using Power BI in a capital markets/investment banking use case.

**3. Tech Stack:**

The dashboard was built using the following tools and technologies:

🔧 Power BI Desktop – Main platform used for dashboard development and interactive reporting.

📊 Power Query – Used for data cleaning, transformation, and shaping of trade-related datasets.

🗃️ Data Modeling – Relationships established among trade, region, counterparty, trader, and risk-related tables to enable filtering and analytical insights.

📈 Data Visualization Techniques – Used KPI cards, donut charts, bar charts, treemaps, and trend analysis visuals for financial storytelling.

📋 File Format – .pbix for development and .png/.pdf for dashboard previews.

**4. Data Source:**

Source: Dummy / simulated investment banking trade dataset created for learning and portfolio purposes only.

The dataset includes:
Trade values,
P&L metrics,

Trade statuses,

Break types,
Trader performance data

⚠️ No real Nomura internal or confidential data has been used in this project.

**5. Features / Highlights:**

Business Problem:
Investment banking operations generate large volumes of trading data across multiple regions, counterparties, and asset classes. Monitoring trade profitability, failed trades, operational breaks, and risk exposure manually can be difficult and time-consuming.

Goal of the Dashboard:
To create a centralized and interactive reporting solution that enables users to:

Monitor trading performance,
Analyze profitability trends,
Track trade lifecycle statuses,
Identify operational trade breaks,
Evaluate risk exposure across asset classes and regions,
Compare trader and counterparty performance
